SmartPlot Portal Detailed Setup Guide

Step-by-step instructions for opening the GitHub repository, cloning it, setting up MySQL, and running the project locally.

Repository Link

https://github.com/pranavburadi/smartplot-portal

1. What This Guide Covers

This document is for anyone who wants to run your project on another laptop or PC after opening the GitHub repository.

It explains:

  • what software needs to be installed first
  • how to clone the project
  • how to create the MySQL database
  • how to run the Spring Boot backend
  • how to open the frontend in the browser
  • what to do if something does not work

2. Requirements

Before running the project, install these tools.

Tool Why It Is Needed Link
Git To clone the GitHub repository https://git-scm.com/downloads
Java JDK To run the Spring Boot application https://adoptium.net/
VS Code To open and run the project easily https://code.visualstudio.com/
Extension Pack for Java Lets VS Code run Java projects properly VS Code Java Pack

Important

You do not need a public deployment link for local usage. This project can be run entirely on a local machine.

3. Step 1: Open the GitHub Repository

  1. Open this link in your browser:
    https://github.com/pranavburadi/smartplot-portal
  2. Click the green Code button.
  3. Make sure HTTPS is selected.
  4. Copy the repository URL.

4. Step 2: Clone the Project

Open terminal, PowerShell, or command prompt and run:

git clone https://github.com/pranavburadi/smartplot-portal.git

Then go into the project folder:

cd smartplot-portal

At this point, the project files should be on the computer.

5. Step 3: Open the Project in VS Code

  1. Open VS Code.
  2. Click File -> Open Folder.
  3. Select the cloned project folder smartplot-portal.
  4. Wait for VS Code to load the Java project.
Main Files to Know
  • pom.xml — project dependencies and build configuration
  • src/main/java/com/example/smartplot/SmartplotApplication.java — application entry point
  • src/main/resources/application.properties — app configuration
  • src/main/resources/static/index.html — frontend page
  • src/main/resources/static/app.js — frontend logic
  • src/main/resources/static/styles.css — frontend styling

6. Step 4: Create the MySQL Database

Open MySQL Workbench or MySQL command line and create the database:

CREATE DATABASE smart_plot_portal;

The project will use this database name locally.

Optional: Check That MySQL Is Running

If MySQL is not running, the Spring Boot app will fail to start.

Typical local config expected by the project:

  • Host: localhost
  • Port: 3306
  • Database: smart_plot_portal
  • User: root
  • Password: root123

7. Step 5: Review Local Configuration

Open this file in the project:

src/main/resources/application.properties

The local fallbacks currently support local development. If the local MySQL setup is different, update the values or set environment variables.

What this file controls:

  • database connection
  • server port
  • JWT secret
  • JPA table update behavior

8. Step 6: Run the Application

Method A: Run with VS Code

  1. Open SmartplotApplication.java
  2. Right-click it
  3. Click Run Java

Method B: Run with Maven

If Maven is installed, run:

mvn spring-boot:run
If Maven Is Not Installed

Use the VS Code Run Java option. That is enough for local execution.

9. Step 7: Open the Frontend

Once Spring Boot starts successfully, open this in the browser:

http://localhost:8081/

If the page looks outdated after code changes, hard refresh:

Ctrl + F5

10. What the User Should See

The portal should show:

  • signup and login area
  • live user section
  • plot inventory section
  • plot cards with price, area, status, and optional image
  • My booked plots section

  1. Open the homepage at http://localhost:8081/

13. Troubleshooting

Problem: Spring Boot does not start

Possible reasons:

  • MySQL is not running
  • Wrong DB username/password
  • Database smart_plot_portal does not exist
  • Java project did not load properly in VS Code

Problem: Browser opens but shows errors

  • Log out and log in again
  • Hard refresh with Ctrl + F5
  • Check terminal for backend error

Problem: Plot images do not show

  • Make sure the URL is public and reachable
  • Try a different image URL

Problem: Java import issues in VS Code

  1. Press Ctrl + Shift + P
  2. Run Java: Clean Java Language Server Workspace
  3. Restart VS Code
