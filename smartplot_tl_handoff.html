Current State

Working local MVP / prototype for real estate plot listing and booking.

1. Executive Summary

SmartPlot Portal is a full-stack real estate portal prototype built using Spring Boot, MySQL, JWT authentication, and a frontend served directly from the backend. It currently supports user registration, login, live plot inventory management, image-based listing display, plot booking, booking cancellation, and a user-specific “My booked plots” section.

The project is functional as a local demo system and is suitable for internal review, academic submission, or continued team development. It is not yet a complete production-grade real estate platform.

2. What the Project Currently Is

The project currently combines two workflows in a single portal:

  • Buyer-facing flow: sign up, log in, browse listings, book available plots, cancel own bookings
  • Internal inventory flow: create plots, edit listings, delete listings, add image URLs, update status
Current Value

This is already a usable end-to-end prototype showing the core booking lifecycle from listing to reservation.

3. Tech Stack

Layer Technology Purpose
Backend Spring Boot 3 REST APIs, business logic, application runtime
Security Spring Security + JWT Authentication and protected endpoints
Persistence Spring Data JPA Database access through repositories
Database MySQL Stores users and plots data
Frontend HTML, CSS, JavaScript User interface, booking flow, dashboard rendering

4. Features Implemented

Authentication

  • User signup
  • User login
  • Password hashing with BCrypt
  • JWT token-based authentication
  • Protected API routes
  • Logout flow
  • Login persistence using browser local storage

Users

  • Basic user directory display
  • View all registered users

Plots / Inventory

  • Create plot listings
  • Edit plot listings
  • Delete plot listings
  • Search plots by plot number, owner, location
  • Filter plots by status
  • Add and display image URLs for plots

Booking

  • Book an available plot as the logged-in user
  • Only AVAILABLE plots can be booked
  • Booking changes plot status to RESERVED
  • Booked plot is linked to the logged-in user by name/email
  • User can view My booked plots
  • User can cancel their own booking

5. Current Business Flow

  1. User signs up or logs in.
  2. Backend returns a JWT token.
  3. Frontend stores the token and uses it in protected requests.
  4. User browses live plot inventory.
  5. User books an available plot.
  6. Plot status changes from available to reserved.
  7. Plot appears under the user’s booking section.
  8. User can cancel the booking and release the plot back to inventory.

8. What Is Still Missing

Important

This project is functional, but it does not yet fully satisfy a complete production-grade real estate portal specification.

Missing or Partial Areas

  • Role-based access: no full separation yet for CLIENT / DEVELOPER / ADMIN
  • Separate dashboards: buyer and admin flows are still combined in one portal
  • Property details page: no dedicated single-property detailed route/page
  • Real image upload: only image URLs are supported, not file uploads
  • Payment module: no mock or live payment system yet
  • Notifications: no email or in-app alert flow
  • Reports / analytics: not implemented
  • Admin approvals: not implemented
  • User profile management: very basic, no edit/delete/profile screen
  • Production deployment: deployment prep exists, but no fully active public hosting link

9. Suggested Next Development Priorities

  1. Role Separation
    Introduce roles like CLIENT, DEVELOPER, ADMIN and secure routes accordingly.
  2. Split Dashboards
    Create one buyer dashboard for browsing/booking and one admin dashboard for listing management.
  3. Property Detail Pages
    Add a dedicated page or modal with more information, images, and booking CTA.
  4. Payment Simulation
    Add a dummy payment success/failure flow for bookings.
  5. Admin Approval Workflow
    Allow new properties to be approved before being visible publicly.
  6. Analytics & Reporting
    Add counts, totals, and booking activity reports.
  7. Deployment
    Host backend and DB so the project is accessible publicly.

10. How to Run It Locally

Requirements

  • Git
  • Java JDK
  • VS Code or IntelliJ
  • MySQL Server

Repository

https://github.com/pranavburadi/smartplot-portal

Clone

git clone https://github.com/pranavburadi/smartplot-portal.git
cd smartplot-portal

Create Database

CREATE DATABASE smart_plot_portal;

Run

Open the project in VS Code and run:

src/main/java/com/example/smartplot/SmartplotApplication.java

Then open:

http://localhost:8081/

11. Current Local Configuration Assumption

The project currently expects a local MySQL setup similar to:

  • Host: localhost
  • Port: 3306
  • Database: smart_plot_portal
  • Username: root
  • Password: root123

These can be overridden with environment variables for deployment.
